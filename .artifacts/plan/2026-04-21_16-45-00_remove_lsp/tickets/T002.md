## Goal

Remove `maybe_prepend_lsp_diagnostics` import and call from `hashline_edit.py` and `write_file.py`.

## Owner

backend

## Time

15m

## Difficulty

easy

## Files

- `src/tunacode/tools/hashline_edit.py`
- `src/tunacode/tools/write_file.py`

## Context

Both file tools currently import `maybe_prepend_lsp_diagnostics` from the deleted `tunacode.tools.lsp.diagnostics` module and await it before returning tool results. After removing LSP, the tools should return plain strings directly.

## Verification

```bash
grep -n "lsp" src/tunacode/tools/hashline_edit.py src/tunacode/tools/write_file.py
```

Should return zero matches.

## Edits

### `src/tunacode/tools/hashline_edit.py`

- Remove the import:
  ```python
  from tunacode.tools.lsp.diagnostics import maybe_prepend_lsp_diagnostics
  ```
- In `_run_hashline_edit`, change the return statement from:
  ```python
  return await maybe_prepend_lsp_diagnostics(output, Path(filepath))
  ```
  to:
  ```python
  return output
  ```

### `src/tunacode/tools/write_file.py`

- Remove the import:
  ```python
  from tunacode.tools.lsp.diagnostics import maybe_prepend_lsp_diagnostics
  ```
- In `_run_write_file`, change the return statement from:
  ```python
  return await maybe_prepend_lsp_diagnostics(result, Path(filepath))
  ```
  to:
  ```python
  return result
  ```
