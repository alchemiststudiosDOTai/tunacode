## Goal

Remove `LspSettings` TypedDict and the `lsp` field from `UserSettings`, defaults, and validation.

backend

## Context

The type system and configuration layer explicitly model LSP settings. Since the feature is removed, these definitions must be excised to prevent stale references and to keep the config schema clean.

Run:

## Edits

### `src/tunacode/types/base.py`

1. Delete the `LspSettings` class:
   ```python
   class LspSettings(TypedDict):
       enabled: bool
       timeout: float
   ```

2. Remove `"lsp": LspSettings` from `UserSettings`.

### `src/tunacode/types/__init__.py`

1. Remove `LspSettings` from the re-export `from tunacode.types.base import ...` block.

### `src/tunacode/configuration/defaults.py`

1. Remove the nested `"lsp"` block from `DEFAULT_USER_CONFIG["settings"]`:
   ```python
   "lsp": {
       "enabled": True,
       "timeout": 5.0,
   },
   ```

### `src/tunacode/configuration/user_config.py`

1. Remove `LspSettings` from the imports at the top.

2. Delete `_validate_lsp_settings`:
   ```python
   def _validate_lsp_settings(value: object) -> LspSettings:
       raw_lsp = _require_mapping(value, path="settings.lsp")
       return LspSettings(
           enabled=_require_bool(raw_lsp["enabled"], path="settings.lsp.enabled"),
           timeout=_require_float(raw_lsp["timeout"], path="settings.lsp.timeout"),
       )
   ```

3. In `_validate_settings`, remove `lsp=_validate_lsp_settings(raw_settings["lsp"])`.
