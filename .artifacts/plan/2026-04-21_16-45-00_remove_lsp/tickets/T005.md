## Context

`repl_support.py` defines `DIAGNOSTICS_BLOCK_START`, `DIAGNOSTICS_BLOCK_END`, `DIAGNOSTICS_BLOCK_PATTERN`, and `DIAGNOSTICS_BLOCK_RE`. These constants and their usage in `build_tool_result_callback` are only relevant for stripping LSP diagnostics blocks from tool results before displaying them. Since LSP is removed, this logic is dead code.

## Verification


Should return zero matches.

## Edits

1. Remove these constant definitions (around lines 40-42):
   ```python
   DIAGNOSTICS_BLOCK_START: str = "<file_diagnostics>"
   DIAGNOSTICS_BLOCK_END: str = "</file_diagnostics>"
   DIAGNOSTICS_BLOCK_PATTERN: str = f"{DIAGNOSTICS_BLOCK_START}.*?{DIAGNOSTICS_BLOCK_END}"
   DIAGNOSTICS_BLOCK_RE = re.compile(DIAGNOSTICS_BLOCK_PATTERN, re.DOTALL)
   ```

2. In `build_tool_result_callback`, remove the diagnostics-match guard logic. The current code around line 159 is:
   ```python
   diagnostics_match = DIAGNOSTICS_BLOCK_RE.match(content)
   if diagnostics_match:
       if content.startswith(DIAGNOSTICS_BLOCK_START):
           # ...
   ```
   Replace the entire block with the simpler direct logic: just take the `TextContent` value as-is without any diagnostics stripping. The exact lines to remove depend on the current implementation, but the goal is to delete anything referencing `DIAGNOSTICS_BLOCK_RE` or `DIAGNOSTICS_BLOCK_START`.
