## Patterns Found
- Extension-based server registry:
  - `src/tunacode/tools/lsp/servers.py:11-60` maps `.py` and `.pyi` to `ruff server`, `.ts/.tsx/.js/.jsx` to `typescript-language-server --stdio`, `.go` to `gopls`, and `.rs` to `rust-analyzer`.
- Shared client cache keyed by workspace root and command:
  - `src/tunacode/tools/lsp/__init__.py:61-72` builds `client_key = f"{root}::{command_key}"`, starts a client on first use, stores it in `_clients`, and reuses it for subsequent calls.
- Workspace root discovery from filesystem markers:
  - `src/tunacode/tools/lsp/__init__.py:30-38` walks parent directories until one contains any `WORKSPACE_MARKERS` entry.
- Minimal JSON-RPC transport over stdio:
  - `src/tunacode/tools/lsp/client.py:66-72` starts the language server with `asyncio.create_subprocess_exec(...)`.
  - `src/tunacode/tools/lsp/client.py:163-179` serializes JSON messages with `Content-Length` framing.
  - `src/tunacode/tools/lsp/client.py:205-236` reads a single JSON-RPC message from stdout.
- Diagnostic collection from `textDocument/publishDiagnostics`:
  - `src/tunacode/tools/lsp/client.py:200-203` routes notifications with method `textDocument/publishDiagnostics`.
  - `src/tunacode/tools/lsp/client.py:238-264` converts raw diagnostic payloads into `Diagnostic` instances and stores them by file URI.
- File-tool prepend flow:
  - `src/tunacode/tools/lsp/diagnostics.py:41-60` conditionally runs LSP diagnostics and prepends a `<file_diagnostics>` block to the tool result.
  - `src/tunacode/tools/hashline_edit.py:259-263` calls `maybe_prepend_lsp_diagnostics()` after writing and diff generation.
  - `src/tunacode/tools/write_file.py:56-57` calls `maybe_prepend_lsp_diagnostics()` after a new file is created.
- UI status flow:
  - `src/tunacode/ui/app.py:795-800` calls `get_lsp_server_info(filepath)` and forwards the result to the resource bar.
  - `src/tunacode/ui/widgets/resource_bar.py:120-125` appends the server name plus `●` or `○` to the top bar when `_lsp_server` is set.
- UI diagnostics rendering flow:
  - `src/tunacode/ui/renderers/tools/hashline_edit.py:79-82` extracts the diagnostics block before parsing the diff.
  - `src/tunacode/ui/renderers/tools/hashline_edit.py:397-412` renders an extra diagnostics zone when a diagnostics block is present.
  - `src/tunacode/ui/renderers/tools/diagnostics.py:40-104` parses `<file_diagnostics>...</file_diagnostics>`.
  - `src/tunacode/ui/renderers/tools/diagnostics.py:107-168` renders inline diagnostics with counts and per-line messages.
